# 杀戮尖塔2 × 英雄联盟 Mod：烬完整角色设计与 88 张卡池

## 0. 版本目标

本设计文档用于实现一个以英雄联盟角色 **烬 / Jhin** 为核心的《杀戮尖塔2》风格 Mod 角色。

设计目标：

- 角色具有明确的“第四枪”节奏。
- 玩家围绕弹匣、华彩、标记、莲花陷阱和谢幕进行规划。
- 卡池总数为 **88 张**。
- 卡牌类型比例约为 **攻击:技能:能力 = 37:25:21**。
- 最终实际整数分配为：
  - 攻击牌：39 张
  - 技能牌：27 张
  - 能力牌：22 张
  - 总计：88 张

# 2. 核心机制

## 2.1 弹匣

烬拥有专属战斗资源：**子弹**。

默认规则：

- 战斗开始时，烬拥有 4 / 4 发子弹。
- 每使用一张带有关键词“射击”的牌，消耗 1 发子弹。
- 当子弹从 1 变为 0 时，触发“华彩”。
- 子弹为 0 时，不能使用射击牌，除非卡牌特殊说明。
- 回合开始时，如果子弹为 0，自动装填至 4。
- 回合开始时，如果子弹不为 0，则保持当前子弹数。

设计目的：

> 玩家不能每回合无脑回满弹匣，而是要决定是否打空、何时打空、用哪一张牌打出第四枪。

## 2.2 射击

**射击**：使用时消耗 1 发子弹。没有子弹时不能使用。

部分射击牌具有额外规则：

- 本牌不消耗子弹。
- 下一张射击不消耗子弹。
- 射击多次，每次都消耗子弹。

如果射击牌没有消耗子弹，则默认不能触发华彩，除非卡牌另有说明。

## 2.3 华彩

**华彩**：如果这张牌消耗的是最后一发子弹，触发额外效果。

华彩触发时：

- 触发卡牌自身的华彩效果。
- 触发角色遗物“低语”的增伤。
- 触发所有监听华彩的能力牌和遗物。
- 记录本场战斗华彩次数。
- 记录本回合华彩次数。

## 2.4 标记

**标记**：被标记的敌人受到烬的下一张射击牌时，每层额外受到 3 点伤害，然后移除所有标记。

规则：

- 非射击攻击不会消耗标记。
- 多段射击时，默认第一段命中会消耗标记。
- 特殊卡牌可以声明“不移除标记”。

## 2.5 莲花陷阱

**莲花陷阱**：敌人攻击后触发。

每层莲花陷阱：

- 敌人攻击后，受到 6 点伤害。
- 敌人获得 1 层虚弱。
- 如果该敌人死亡，对所有敌人造成 4 点伤害。

建议实现为怪物身上的 Debuff。

## 2.6 谢幕

**谢幕**：强力终结机制。通常要求子弹为空，或本回合触发过华彩。

谢幕牌的共同设计原则：

- 条件较严格。
- 伤害较高。
- 常带有消耗。
- 更适合残局、Boss 战或多敌收割。

# 3. 初始遗物

## 低语

**低语**  
角色专属初始遗物

效果：

- 战斗开始时获得 4 发子弹。
- 每当你触发华彩，造成的攻击伤害提高 50%。
- 如果目标生命低于 25%，额外造成 6 点伤害。

实现建议：

- 华彩伤害先结算卡牌基础伤害和卡牌华彩加成。
- 再乘以 1.5。
- 最后判断目标是否低于 25% 生命，若是则 +6。
- 小数向下取整。

# 4. Build 体系

## 4.1 第四枪爆发流

核心关键词：射击、华彩、第四枪、爆发。

核心牌：

- 第四乐章
- 最后一幕
- 第三声枪响
- 完美换弹
- 完美命中
- 华彩节奏
- 低语回响

玩法：

通过控制弹匣数量，让关键高伤害攻击吃到华彩和低语加成。

优点：

- 单体爆发极高。
- 精英和 Boss 战表现优秀。

缺点：

- 需要规划。
- 失误时容易把第四枪打在低价值牌上。

## 4.2 标记狙杀流

核心关键词：标记、远距狙击、死亡预告、镜头锁定。

核心牌：

- 瞄准射击
- 低声威胁
- 观察弱点
- 死亡预告
- 镜头锁定
- 远距狙击
- 名作诞生
- 终幕伏笔

玩法：

堆叠标记，再使用高倍率射击或华彩射击引爆。

优点：

- 单体稳定。
- 构筑成长明确。

缺点：

- 启动偏慢。
- 多敌战需要配合 AoE 或标记转移。

## 4.3 莲花陷阱流

核心关键词：莲花陷阱、虚弱、引爆、死亡爆炸。

核心牌：

- 设下陷阱
- 盛大布景
- 暗藏机关
- 引爆莲花
- 莲花工坊
- 致命舞台

玩法：

通过给敌人埋设陷阱，利用敌人攻击触发伤害和虚弱，再用引爆或击杀触发 AoE。

优点：

- 防守能力强。
- 多敌战表现好。

缺点：

- 遇到不攻击的敌人会变弱。
- 需要控制触发时机。

## 4.4 谢幕终结流

核心关键词：谢幕、残血、斩杀、收割。

核心牌：

- 谢幕
- 精准谢幕
- 谢幕准备
- 谢幕余音
- 华丽处刑
- 死亡是艺术
- 终幕艺术
- 完美犯罪

玩法：

前期压低敌人血线，后期用谢幕或斩杀牌收割。

优点：

- 残局极强。
- 多敌收割能力优秀。

缺点：

- 依赖血线判断。
- 容易过度追求斩杀而忽视防御。

## 4.5 装填循环流

核心关键词：装填、子弹、抽牌、格挡、循环。

核心牌：

- 装填
- 冷静装填
- 完美装填
- 完美换弹
- 枪械保养
- 演出计划
- 华彩节奏

玩法：

通过装填获取格挡、抽牌、额外伤害或能量，形成稳定循环。

优点：

- 稳定性高。
- 防御和资源能力较好。

缺点：

- 需要严格防止无限。
- 装填收益过高会破坏第四枪节奏。

## 8.4 特殊灵魂卡 2 张

灵魂卡不进入普通 88 张卡池统计。

定位：

- 灵魂卡是通过特殊事件把指定普通卡升格而来。
- 它们不应在普通奖励、商店、随机生成池中出现。
- 每局最多获得 1 张灵魂卡，除非后续专门设计高风险事件。
- 灵魂卡不再能通过普通营火升级，因为它们已经属于“升格形态”。

参考原则：

- 参考《杀戮尖塔2》中“先古卡”的设计思路：由特殊奖励把初始或基础牌转化成强力特殊版本。
- 灵魂卡应该明显强于普通卡，但不能完全无条件强，要保留烬的节奏主题。
- 灵魂卡应强化角色核心机制，而不是单纯堆数值。

### 灵魂事件：残响剧场

事件名：残响剧场

触发建议：

- 第二层或第三层特殊事件。
- 只能在烬角色出现。
- 牌组中必须拥有至少 1 张可升格的指定普通卡。

事件文本示例：

> 剧场空无一人，只有聚光灯照在舞台中央。  
> 你听见自己的枪声在幕布后反复回响。  
> 它们不像记忆，更像灵魂。  
> 这一次，演出不再需要观众。

选项：

1. 让子弹歌唱：将 1 张“普通射击”转化为“灵魂射击”。失去 6 点最大生命。
2. 让弹匣呼吸：将 1 张“装填”转化为“终幕装填”。失去 6 点最大生命。
3. 离开：无事发生。

限制：

- 如果牌组中没有普通射击，则选项 1 不出现。
- 如果牌组中没有装填，则选项 2 不出现。
- 如果两者都没有，则事件只显示离开。

### S1. 灵魂射击

由“普通射击”通过特殊事件升格而来。

攻击 / 灵魂  
费用：1

效果：

- 射击。
- 造成 14 点伤害。
- 给予目标 2 层标记。
- 如果这是华彩，改为造成 28 点伤害，给予目标 4 层标记，并施加 3 层易伤。
- 如果本牌击杀敌人，恢复 1 发子弹。

特殊规则：

- 本牌视为“普通射击”的灵魂版本。
- 不进入普通卡池。
- 不能通过营火升级。
- 可以被复制，但复制出的牌仍然是灵魂牌。

设计目的：

- 把初始打击牌变成真正的第四枪核心。
- 同时支持标记流、第四枪爆发流和谢幕终结流。
- 击杀后恢复子弹，制造烬式连续收割，但需要击杀才能触发。

平衡风险：

- 与低语、终幕艺术、华彩节奏叠加后爆发很高。
- 建议每场战斗中“击杀恢复子弹”最多触发 1 次，或者先保留无限制后重点测试。

### S2. 终幕装填

由“装填”通过特殊事件升格而来。

技能 / 灵魂  
费用：0

效果：

- 将子弹恢复至 4。
- 抽 3 张牌。
- 本回合下一张射击牌必定触发华彩。
- 该射击牌造成的伤害提高 25%。
- 消耗。

特殊规则：

- 本牌视为“装填”的灵魂版本。
- 不进入普通卡池。
- 不能通过营火升级。
- 本牌不会关闭华彩；相反，它强制安排一次华彩。

设计目的：

- 把普通装填从容错牌变成一次完整爆发回合的启动器。
- 适合第四枪爆发流、谢幕终结流、单体 Boss 流。
- 抽 3 张牌保证玩家有机会找到关键射击牌。

平衡风险：

- 与最后一幕、完美命中、最后的观众组合时可能打出极高爆发。
- 因为本牌消耗，且需要事件代价，所以允许它明显强于普通卡。
- 如测试过强，可将“伤害提高 25%”改为“额外造成 8 点伤害”。

# 11. UI 与反馈设计

## 11.1 弹匣 UI

显示在角色血条附近。

示例：

- 有弹：●
- 空弹：○

4 发满弹：

> ● ● ● ●

剩 1 发：

> ○ ○ ○ ●

剩 1 发时最后一颗子弹应高亮，提示下一张射击会触发华彩。

## 11.2 卡牌提示

如果当前子弹为 1：

- 可触发华彩的射击牌边框变为金色。
- Tooltip 显示：“这张牌将触发华彩。”

如果当前子弹为 0：

- 普通射击牌置灰。
- Tooltip 显示：“没有子弹，无法使用。”

## 11.3 华彩反馈

触发华彩时建议包含：

- 屏幕短暂暗化。
- 金色或紫色枪焰。
- 较重的枪声音效。
- 伤害数字加粗或高亮。
- 敌人残血时增加斩杀音效。

## 11.4 谢幕反馈

使用谢幕类卡牌时建议：

- 镜头拉远或屏幕压暗。
- 出现四道狙击线。
- 每次射击有独立音效。
- 第四次射击音效最重。

# 12. 实现结构建议

## 12.1 角色字段

建议在角色或专属 Power 中维护：

```java
int bullets;
int maxBullets;
boolean canTriggerFlourish;
int flourishCountThisCombat;
int flourishCountThisTurn;
boolean usedShootThisTurn;
boolean nextShotNoBulletCost;
int nextShotBonusDamage;
```

默认值：

```java
bullets = 4;
maxBullets = 4;
canTriggerFlourish = true;
flourishCountThisCombat = 0;
flourishCountThisTurn = 0;
usedShootThisTurn = false;
nextShotNoBulletCost = false;
nextShotBonusDamage = 0;
```

## 12.2 战斗开始

```java
onBattleStart() {
    maxBullets = 4;
    bullets = maxBullets;
    canTriggerFlourish = true;
    flourishCountThisCombat = 0;
    flourishCountThisTurn = 0;
    usedShootThisTurn = false;
}
```

## 12.3 回合开始

```java
atStartOfTurn() {
    flourishCountThisTurn = 0;
    usedShootThisTurn = false;
    canTriggerFlourish = true;

    if (bullets == 0) {
        reload();
    }
}
```

## 12.4 装填

```java
reload() {
    bullets = maxBullets;
    notifyReload();
}
```

## 12.5 射击结算

```java
useShootCard(AbstractMonster target, ShootCard card) {
    usedShootThisTurn = true;

    boolean consumedBullet = false;
    boolean isFlourish = false;

    if (nextShotNoBulletCost) {
        nextShotNoBulletCost = false;
    } else {
        if (bullets <= 0) {
            cannotUseCard();
            return;
        }

        bullets--;
        consumedBullet = true;

        if (bullets == 0 && canTriggerFlourish) {
            isFlourish = true;
        }
    }

    int damage = card.baseDamage;

    damage += nextShotBonusDamage;
    nextShotBonusDamage = 0;

    if (isFlourish) {
        card.triggerFlourishEffect(target);
        damage = applyWhisperBonus(damage, target);
        flourishCountThisCombat++;
        flourishCountThisTurn++;
        notifyFlourishTriggered(card, target);
    }

    dealDamage(target, damage);
}
```

# 16. 平衡测试指标

每局记录：

- 第一层精英胜率
- 第一层 Boss 胜率
- 平均每场华彩触发次数
- 平均每回合射击次数
- 平均每场装填次数
- 平均每场标记额外伤害
- 平均每场莲花陷阱伤害
- 谢幕平均伤害
- 是否出现无限循环
- 是否出现一回合击杀 Boss
- 是否出现防御不足

健康区间建议：

- 普通战斗：每场 1 到 3 次华彩
- 精英战：每场 3 到 6 次华彩
- Boss 战：每场 6 到 12 次华彩

如果华彩过多：

- 削弱装填牌。
- 减少 0 费射击。
- 降低抽牌能力。
- 降低华彩触发后的能量收益。

如果华彩过少：

- 增加调整子弹的 0 费牌。
- 提高装填收益。
- 强化弹匣 UI 提示。
- 给普通牌更多“子弹为 1 时”的奖励。

# 17. 当前设计中需要重点测试的高风险组合

## 17.1 华彩节奏 + 低语回响

风险：

- 华彩后重复攻击。
- 同时获得能量和抽牌。
- 容易形成爆发过高。

建议：

- 重复牌不能再次触发华彩。
- 重复牌不能恢复子弹。

## 17.2 死亡是艺术 + 枪下谢礼 + 谢幕

风险：

- 击杀后恢复子弹、抽牌、继续攻击。
- 多敌战容易滚雪球。

建议：

- 击杀恢复子弹最多不超过 maxBullets。
- 一些击杀收益牌添加每回合次数限制。

## 17.3 完美装填 + 谢幕准备

风险：

- 抽牌、免费射击、装填同时出现。
- 可能制造一回合过多射击。

建议：

- 谢幕准备降低本回合射击伤害。
- 完美装填消耗。

## 17.4 戏命师 + 四的美学

风险：

- 第 4 张牌、第 8 张牌和第 4 次华彩叠加后收益过高。

建议：

- 保留作为稀有 Build 核心，但测试数值。
- 必要时将戏命师费用保持 3，不给费用升级。

# 18. 最终设计原则

烬的核心不是单纯高伤害，而是：

1. 玩家知道第四枪即将到来。
2. 玩家能决定第四枪打哪张牌。
3. 玩家可以为了更好的第四枪主动等待。
4. 第四枪打出时必须有强烈反馈。
5. 敌人越残，烬越危险。
6. 装填是节奏转换，而不是免费续航。
7. 标记和陷阱负责铺垫，华彩和谢幕负责收束。

最重要的开发优先级：

1. 弹匣 UI
2. 射击消耗子弹
3. 华彩触发与反馈
4. 标记系统
5. 莲花陷阱系统
6. 谢幕演出感
7. 20 张 MVP 卡
8. 完整 88 张卡池
