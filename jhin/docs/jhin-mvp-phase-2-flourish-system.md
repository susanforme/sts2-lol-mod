# Jhin MVP 文档 - 阶段 2：华彩系统

## 1. 阶段目标

本阶段目标是实现烬最核心的爆发机制：当最后一发子弹被消耗时，触发“华彩”。

本阶段重点不是扩展完整 Build，而是把“最后一发子弹 -> 华彩触发 -> 卡牌强化 -> 系统监听 -> 初始遗物增伤”这条核心链路完整打通。

## 2. 本阶段交付内容

必须交付以下内容：

1. 当子弹从 1 变为 0 时，触发华彩。
2. 华彩触发时执行卡牌自身华彩效果。
3. 华彩触发时记录本回合华彩次数。
4. 华彩触发时记录本场战斗华彩次数。
5. 华彩触发时通知遗物、能力牌、Power。
6. 实现初始遗物 `低语` 的核心效果。

## 3. 功能需求

## 3.1 华彩触发条件

需要完成：

- 在射击牌结算过程中判断是否触发华彩。
- 仅当一张射击牌真实消耗了最后一发子弹时触发华彩。
- 判断条件必须基于“子弹从 1 变为 0”的过程，而不是仅检查结算后的结果。

最低要求：

- 当前子弹为 1，且本次射击实际消耗 1 发子弹时，触发华彩。
- 当前子弹大于 1 时，不触发华彩。
- 当前子弹为 0 时本来就无法使用普通射击牌，因此不会出现伪华彩。

## 3.2 华彩触发后的卡牌效果

需要完成：

- 华彩触发时，执行当前卡牌自己的华彩效果。
- 卡牌普通效果与华彩附加效果必须能够区分。
- 同一张射击牌在普通状态与华彩状态下应有不同结算分支。

最低要求：

- `AbstractShootCard` 或等效结构中具备明确的华彩扩展接口。
- 具体卡牌能实现“普通效果 + 华彩额外效果”的组合结算。

## 3.3 华彩计数

需要完成：

- 华彩成功触发时，`flourishCountThisTurn` 加 1。
- 华彩成功触发时，`flourishCountThisCombat` 加 1。
- 回合开始时正确重置 `flourishCountThisTurn`。
- 战斗开始时正确重置 `flourishCountThisCombat`。

最低要求：

- 只有真正成功触发华彩时才计数。
- 非最后一发、未消耗子弹、无法打出的射击均不能错误计数。

## 3.4 华彩事件通知链路

需要完成：

- 华彩触发后向相关系统发送统一事件通知。
- 至少支持遗物、能力牌、Power 对华彩事件进行监听。
- 监听接口应尽量统一，避免未来每加一个效果就改一遍射击主流程。

最低要求：

- `低语` 能基于该通知正确生效。
- 未来能力牌和 Power 可以通过统一入口挂接华彩监听。

## 3.5 不消耗子弹的射击与华彩关系

需要完成：

- 明确区分“射击牌”和“消耗子弹的射击牌”。
- 默认情况下，不消耗子弹的射击不触发华彩。
- 本阶段只实现默认规则，不要求实现“特例允许不耗弹也触发华彩”的扩展卡。

最低要求：

- 即使是一张射击牌，只要本次没有消耗子弹，就不能触发华彩。
- 不允许通过免费射击、复制射击或占位实现误触发华彩。

## 4. 初始遗物：低语

本阶段必须实现 `低语` 的核心效果。

## 4.1 低语的功能需求

需要完成：

- 战斗开始时获得 4 发子弹。
- 当华彩触发时，本次攻击伤害提高 50%。
- 如果目标生命低于 25%，额外造成 6 点伤害。

## 4.2 低语的结算要求

需要遵循以下顺序：

1. 先计算卡牌基础伤害。
2. 再叠加卡牌自身华彩增加的伤害或卡牌效果中的直接增伤。
3. 然后将华彩攻击伤害乘以 1.5。
4. 最后判断目标生命是否低于 25%，若满足则额外 +6。
5. 小数向下取整。

最低要求：

- `低语` 只放大华彩攻击，不放大普通攻击。
- 低血线补伤应在乘算之后附加。
- 不会对非攻击效果、非华彩效果、非伤害效果误加成。

## 5. 建议实现的类与扩展点

本阶段建议优先完善以下类或入口：

- `AbstractShootCard`
- `ShootAction`
- `BulletPower`
- `WhisperRelic` 或等效初始遗物类
- 华彩通知接口，例如 `OnFlourishTriggered`、`NotifyFlourishTriggered` 或等效入口

## 5.1 AbstractShootCard

职责建议：

- 承载射击牌的基础流程。
- 区分普通射击与华彩射击。
- 提供华彩附加效果的可重写接口。

本阶段最低要求：

- 至少具备一个明确的“是否触发华彩”的执行分支。
- 具体卡牌不需要重复编写通用华彩判定。

## 5.2 ShootAction

职责建议：

- 在处理射击时判断是否真实消耗了最后一发子弹。
- 统一驱动华彩触发、计数更新和通知派发。

本阶段最低要求：

- 将“消耗子弹”和“触发华彩”放在同一条可靠结算链路中。
- 避免在不同卡牌里分别写华彩计数和监听通知。

## 5.3 BulletPower

职责建议：

- 作为弹匣系统当前状态的展示和同步入口。
- 辅助体现当前弹药与华彩触发时机。

本阶段最低要求：

- 状态同步稳定。
- 为后续 UI 和 tooltip 扩展提供可读状态。

## 5.4 WhisperRelic

职责建议：

- 响应华彩事件。
- 对华彩攻击伤害进行最终修正。
- 在战斗开始时确保角色拥有基础子弹。

本阶段最低要求：

- 能正确响应华彩。
- 能正确提升华彩伤害。
- 不会错误修改非华彩攻击。

## 6. 非目标

以下内容不属于本阶段：

- 标记系统
- 莲花陷阱系统
- 谢幕系统
- 华彩专属 UI 高亮与完整演出反馈
- 所有能力牌与遗物的完整监听实现
- “不消耗子弹但允许触发华彩”的特殊规则扩展
- 单元测试

## 7. 推荐最小实现策略

建议按以下顺序实现：

1. 在 `ShootAction` 中确定“子弹从 1 变为 0”这一唯一华彩入口。
2. 将华彩状态传回卡牌或卡牌结算逻辑，执行华彩附加效果。
3. 在统一入口更新本回合和本场战斗计数。
4. 在统一入口派发华彩通知。
5. 最后接入 `低语`，完成华彩伤害放大。

重点是建立一条可复用、可监听、不会误触发的华彩主链路。

## 8. 验收标准

本阶段完成后，必须满足以下验收条件：

1. 第四枪可以正确触发华彩。
2. 非最后一发子弹不会触发华彩。
3. 不消耗子弹的射击默认不触发华彩。
4. `低语` 能正确放大华彩伤害。

建议补充手动验证场景：

1. 当前子弹为 2 时打出射击牌，不应触发华彩。
2. 当前子弹为 1 时打出射击牌，应触发华彩。
3. 使用不消耗子弹的射击牌时，不应触发华彩计数。
4. 有无 `低语` 时，对同一张华彩攻击的伤害差值应符合规则。
5. 目标生命低于 25% 时，`低语` 应在华彩加成后额外补 6 点伤害。

## 9. 完成定义

当以下条件全部满足时，可判定“阶段 2 完成”：

- 华彩触发条件已稳定实现。
- 华彩计数正确。
- 华彩通知链路已建立。
- 卡牌可以在华彩时执行额外效果。
- `低语` 能正确响应并放大华彩伤害。
- 实战中不存在误触发、漏触发或重复触发的明显错误。

## 10. 下一阶段接口预留

为了减少后续返工，本阶段应预留以下扩展点：

- 华彩触发后的 UI 高亮与反馈入口
- 能力牌监听华彩后的收益效果入口
- 遗物监听华彩后的额外资源奖励入口
- 强制华彩、禁止华彩、伪华彩等特殊规则扩展点
- 与标记系统、谢幕系统联动的伤害修正入口

这部分只要求结构可扩展，不要求本阶段实现完整内容。
