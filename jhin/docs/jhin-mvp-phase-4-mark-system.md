# Jhin MVP 文档 - 阶段 4：标记系统

## 1. 阶段目标

本阶段目标是实现烬的第二条主要 Build 轴：标记狙杀流。

本阶段重点是打通“施加标记 -> 射击命中 -> 额外伤害结算 -> 标记移除”的完整链路，让标记能够稳定服务于单体爆发和第四枪收束。

## 2. 本阶段交付内容

必须交付以下内容：

1. 实现 `MarkPower`。
2. 实现标记的基础结算规则。
3. 实现 `ApplyMarkAction`。
4. 将标记系统接入首批 MVP 卡牌。

本阶段需要接入的 MVP 卡牌：

- `瞄准射击`
- `布置舞台`
- `远距狙击`
- `致命华彩`

## 3. 标记规则需求

需要完整实现以下规则：

1. 被标记敌人受到烬的下一张射击牌时，每层额外受到 3 点伤害。
2. 命中后移除所有标记。
3. 非射击攻击不消耗标记。
4. 多段射击默认第一段消耗标记。

## 4. 功能需求

## 4.1 MarkPower

需要完成：

- 在敌人身上实现 `MarkPower` 或等效 Debuff。
- 记录当前标记层数。
- 能被卡牌、Action 和射击结算逻辑访问。
- 正确显示层数与说明文本。

最低要求：

- 标记层数叠加稳定。
- 标记可被射击结算正确消费。
- 标记被消耗后会被正确清空。

## 4.2 标记伤害结算

需要完成：

- 当敌人受到烬的射击牌命中时，先读取标记层数。
- 每层标记额外造成 3 点伤害。
- 默认在该次射击命中后移除全部标记。
- 该附加伤害必须稳定进入本次射击的伤害流程。

最低要求：

- `1` 层标记额外造成 `3` 点伤害。
- `2` 层标记额外造成 `6` 点伤害。
- `N` 层标记按 `N * 3` 线性结算。
- 标记附加伤害必须与射击牌绑定，而不是在敌人受击时对所有攻击通用生效。

## 4.3 标记消耗规则

需要完成：

- 只有射击牌命中时才消耗标记。
- 非射击攻击命中时，标记保留不变。
- 若射击牌为多段攻击，默认由第一段命中结算并消耗标记。

最低要求：

- 普通攻击牌、技能牌、非射击伤害来源都不能误消耗标记。
- 多段射击不会重复多次消费同一批标记。
- 标记不会出现“先移除后未加伤”或“已加伤但未移除”的状态错乱。

## 4.4 ApplyMarkAction

需要完成：

- 创建 `ApplyMarkAction`，统一处理标记施加。
- 支持单体施加标记。
- 支持随机目标施加标记或由卡牌控制目标施加标记。

最低要求：

- 施加标记逻辑不要散落在每张卡牌中。
- 所有 MVP 标记牌优先通过 `ApplyMarkAction` 接入。

## 5. MVP 卡牌接入要求

本阶段必须至少完成以下卡牌与标记系统的联动。

## 5.1 瞄准射击

需求：

- 属于射击牌。
- 造成伤害。
- 给予目标标记。

接入重点：

- 要验证“先命中再施加标记”的实际顺序是否符合设计意图。
- 这张牌本身施加的标记不应该被自己本次命中立即消费，除非你后续明确调整结算顺序。

## 5.2 布置舞台

需求：

- 提供格挡。
- 给予随机敌人标记。

接入重点：

- 验证随机目标施加逻辑正确。
- 验证施加标记后，后续射击能正确引爆。

## 5.3 远距狙击

需求：

- 属于高伤害射击牌。
- 与目标标记层数联动。

接入重点：

- 需要明确“卡牌文本自身按标记增伤”和“标记规则每层 +3 伤害”两者都能同时生效。
- 必须避免重复实现导致数值翻倍错误。

## 5.4 致命华彩

需求：

- 属于射击牌。
- 若目标有标记，则施加易伤。
- 华彩时施加更高层数的易伤。

接入重点：

- 需要验证标记是否在判断“目标是否有标记”时仍然存在。
- 应明确“标记判定时机”和“标记消耗时机”，避免先移除再无法判定有标记。

## 6. 关键结算时序要求

本阶段应尽量统一标记相关时序，避免后续出现大量例外判断。

推荐遵循以下顺序：

1. 射击牌命中目标。
2. 判断目标当前是否有标记。
3. 计算标记附加伤害。
4. 结算本次射击与标记联动效果。
5. 若本次为可消耗标记的射击，则移除全部标记。

这样可以保证：

- `致命华彩` 能判断目标“是否有标记”。
- `远距狙击` 能读取命中时的标记层数。
- 标记消费发生在命中结算之后，不会提前丢失状态。

## 7. 建议实现的类与入口

本阶段建议优先完善以下类或接口：

- `MarkPower`
- `ApplyMarkAction`
- `ShootAction`
- `AbstractShootCard`
- 与伤害计算相关的标记读取入口

## 7.1 MarkPower

职责建议：

- 管理标记层数。
- 提供读取、叠加、清空接口。
- 对 UI 和 tooltip 提供标准说明。

本阶段最低要求：

- 层数表现稳定。
- 可被射击主流程准确读取和清空。

## 7.2 ApplyMarkAction

职责建议：

- 统一施加标记。
- 作为所有标记来源卡牌的标准入口。

本阶段最低要求：

- 施加逻辑集中。
- 不因不同卡牌实现方式不同而产生层数异常。

## 7.3 ShootAction

职责建议：

- 在射击命中时读取并结算标记伤害。
- 控制标记消费的准确时机。

本阶段最低要求：

- 射击行为与标记消费绑定在同一条主流程里。
- 不允许由敌方被动受击逻辑去猜测是否该消耗标记。

## 8. 非目标

以下内容不属于本阶段：

- 标记转移系统
- 标记不移除特例
- 莲华陷阱系统
- 谢幕系统
- 完整标记 UI 强化表现
- 单元测试

说明：

- 如果后续要做如 `镜头锁定` 这种“不移除标记”的特殊牌，本阶段只需要预留扩展点，不强制现在实现。

## 9. 推荐最小实现策略

建议按以下顺序实现：

1. 先完成 `MarkPower` 与 `ApplyMarkAction`。
2. 在 `ShootAction` 中统一接入标记读取、加伤和消费。
3. 先打通一张最简单的施加标记牌与一张消耗标记牌。
4. 再补齐 `瞄准射击`、`布置舞台`、`远距狙击`、`致命华彩`。
5. 最后回归“高标记 + 华彩射击”的爆发链路。

重点是先保证规则稳定，再扩展卡牌数量。

## 10. 验收标准

本阶段完成后，必须满足以下验收条件：

1. 标记只被射击牌消耗。
2. 标记伤害正确结算。
3. 高标记 + 华彩射击能形成明显爆发。

建议补充手动验证场景：

1. 对敌人施加 2 层标记后，用非射击攻击命中，标记应保留。
2. 对敌人施加 2 层标记后，用射击牌命中，应额外造成 6 点伤害并清空标记。
3. 对敌人施加高层标记后，用 `远距狙击` 命中，应同时吃到卡牌自带标记联动和标记规则加伤。
4. 用 `致命华彩` 命中带标记目标时，应能正确判定目标有标记并施加易伤。
5. 当高标记目标被第四枪命中时，应能观察到明显爆发提升。

## 11. 完成定义

当以下条件全部满足时，可判定“阶段 4 完成”：

- `MarkPower` 已可用。
- 标记施加、读取、加伤、清空链路完整。
- 射击与非射击对标记的差异处理正确。
- 首批 MVP 标记卡牌已接入并可运行。
- 标记 Build 已具备基础可玩性和爆发辨识度。

## 12. 下一阶段接口预留

为了减少后续返工，本阶段应预留以下扩展点：

- “不移除标记”特例入口
- 标记转移入口
- 标记与易伤、虚弱、谢幕联动的扩展点
- 多敌场景下标记优先级与 AI 目标提示入口

这部分只要求结构可扩展，不要求本阶段实现完整内容。
