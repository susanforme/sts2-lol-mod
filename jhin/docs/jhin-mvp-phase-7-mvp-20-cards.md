# Jhin MVP 文档 - 阶段 7：MVP 卡牌 20 张

## 1. 阶段目标

本阶段目标是先完成最小可玩卡池，验证烬的基础循环是否成立。

本阶段重点不是一次性补完 88 张卡池，而是先用 20 张 MVP 卡牌搭起可战斗、可构筑、可验证核心机制的最小版本，让第四枪、标记、莲华陷阱、谢幕和装填循环都能被实际体验。

## 2. 本阶段交付内容

必须交付以下 20 张 MVP 卡牌，并按设计文档定义接入到可运行战斗流程中：

1. `普通射击`
2. `优雅走位`
3. `装填`
4. `致命华彩`
5. `精准一枪`
6. `低语点射`
7. `瞄准射击`
8. `穿透弹`
9. `曼舞手雷`
10. `收尾`
11. `布置舞台`
12. `精密计算`
13. `后撤步`
14. `冷静装填`
15. `设下陷阱`
16. `远距狙击`
17. `第四乐章`
18. `盛大布景`
19. `华彩节奏`
20. `谢幕`

## 3. 功能目标

这 20 张卡必须共同支撑以下最小玩法体验：

- 第四枪爆发
- 标记增伤
- 莲华陷阱
- 谢幕终结
- 装填循环

## 4. 实现范围要求

本阶段每张卡牌都需要具备：

- 正确的费用、类型、稀有度
- 正确的基础效果
- 正确的升级效果
- 正确的关键词归类
- 正确的本地化文本
- 正确的卡图占位资源接入

资源要求：

- 所有卡牌图片默认使用 `JhinMod\Images\card_placeholder.png`。

最低要求：

- 20 张卡都能被创建、抽取、打出、升级。
- 20 张卡都不会因缺少文本、资源、逻辑挂接而导致战斗崩溃。

## 5. 卡牌分组实现建议

为了降低联调成本，建议按功能分组逐步实现，而不是机械按 1 到 20 顺序全部平铺。

## 5.1 初始卡组与基础循环组

优先级最高，先保证角色开局能玩：

1. `普通射击`
2. `优雅走位`
3. `装填`
4. `致命华彩`

目标：

- 先打通初始卡组。
- 验证基础攻击、防御、装填和华彩链路。

## 5.2 基础射击强化组

用于补齐第四枪节奏与普通输出曲线：

1. `精准一枪`
2. `低语点射`
3. `穿透弹`
4. `第四乐章`

目标：

- 验证不同费率、AoE、第四枪限制牌的基本工作是否正常。

## 5.3 标记组

用于验证标记狙杀轴：

1. `瞄准射击`
2. `布置舞台`
3. `远距狙击`
4. `致命华彩`

目标：

- 验证施加标记、保留标记、按层增伤、华彩联动是否成立。

说明：

- `致命华彩` 同时属于初始卡组与标记轴关键卡，应在两个阶段都回归验证。

## 5.4 防守与循环组

用于补足节奏控制、抽牌与装填循环：

1. `精密计算`
2. `后撤步`
3. `冷静装填`
4. `华彩节奏`

目标：

- 验证抽牌、格挡、装填、能量收益和回合循环是否能形成闭环。

## 5.5 收尾与多敌处理组

用于补足残局与战场结构：

1. `曼舞手雷`
2. `收尾`
3. `谢幕`

目标：

- 验证非射击收尾、低血增伤、谢幕终结是否成立。

## 5.6 莲华陷阱组

用于验证多敌战与反击收益：

1. `设下陷阱`
2. `盛大布景`

目标：

- 验证单体陷阱、全体陷阱和敌人攻击后触发链路。

## 6. 单卡实现要求

以下内容适用于本阶段全部 20 张卡：

1. 能在牌库、手牌、弃牌堆、抽牌堆之间正常流转。
2. 能在升级后正确更新数值与描述。
3. `射击` 牌必须接入统一射击结算逻辑。
4. `装填` 牌必须接入统一装填逻辑。
5. `华彩` 牌必须接入统一华彩判定逻辑。
6. 标记与陷阱相关牌必须走统一 Action / Power 实现。

## 7. 关键联调要求

本阶段核心不是“20 张卡都能单独打出”，而是“20 张卡放在一起后循环成立”。

必须重点联调以下组合：

## 7.1 第四枪爆发

建议回归组合：

- `普通射击`
- `精准一枪`
- `低语点射`
- `第四乐章`
- `致命华彩`

验证目标：

- 玩家能够明确规划第四枪。
- 第四枪收益显著高于普通射击。

## 7.2 标记增伤

建议回归组合：

- `瞄准射击`
- `布置舞台`
- `远距狙击`
- `致命华彩`

验证目标：

- 标记叠加后能明显提升单次射击爆发。

## 7.3 莲华陷阱

建议回归组合：

- `设下陷阱`
- `盛大布景`

验证目标：

- 多敌战中能稳定触发攻击后伤害与爆炸收益。

## 7.4 谢幕终结

建议回归组合：

- `装填`
- `冷静装填`
- `第四乐章`
- `谢幕`

验证目标：

- 空弹后的残局终结逻辑成立。

## 7.5 装填循环

建议回归组合：

- `装填`
- `冷静装填`
- `精密计算`
- `后撤步`
- `华彩节奏`

验证目标：

- 装填不会变成无脑回满的免费续航。
- 循环存在收益，但不会立刻失控。

## 8. 建议实现的公共依赖

本阶段卡牌大量依赖前面阶段的公共系统，必须确保以下底层已稳定：

- `AbstractJhinCard`
- `AbstractShootCard`
- `ShootAction`
- `ReloadAction`
- `BulletPower`
- `MarkPower`
- `LotusTrapPower`
- `ApplyMarkAction`
- `ApplyLotusTrapAction`
- `低语`

如果这些基础模块未稳定，本阶段卡牌联调成本会显著升高。

## 9. 非目标

以下内容不属于本阶段：

- 完整 88 张卡池实现
- 稀有 Build 的完整平衡优化
- 灵魂卡系统
- 所有遗物与药水的完整接入
- 高级特效与音频打磨
- 单元测试

说明：

- 本阶段只要求“最小可玩”，不是“完整毕业版本”。

## 10. 推荐最小实现策略

建议按以下顺序推进：

1. 先完成初始卡组 4 张。
2. 再补基础射击强化与基础循环牌。
3. 接入标记系统相关卡。
4. 接入陷阱系统相关卡。
5. 最后补 `谢幕` 与 `华彩节奏` 这类对系统完整度要求更高的牌。
6. 全量回归 20 张卡混合战斗表现。

重点是逐组闭环，而不是一次写完全部卡再集中排错。

## 11. 验收标准

本阶段完成后，必须满足以下验收条件：

1. 初始卡组可正常战斗。
2. 基础攻击、防御、抽牌、装填都可用。
3. 至少能体验到：
   - 第四枪爆发
   - 标记增伤
   - 莲华陷阱
   - 谢幕终结
   - 装填循环

建议补充手动验证场景：

1. 用初始卡组进入普通战斗，不崩溃且能正常完成若干回合。
2. 用 `第四乐章` 或 `致命华彩` 打出明确的第四枪收益。
3. 用 `瞄准射击 + 远距狙击` 打出标记爆发。
4. 用 `设下陷阱 + 盛大布景` 在多敌战中触发陷阱收益。
5. 用 `装填 / 冷静装填 / 精密计算 / 后撤步 / 华彩节奏` 验证资源循环。
6. 在空弹情况下打出 `谢幕`，完成残局终结。

## 12. 完成定义

当以下条件全部满足时，可判定“阶段 7 完成”：

- 20 张 MVP 卡牌已全部可用。
- 初始卡组与扩展卡牌能共同组成可运行的最小卡池。
- 第四枪、标记、陷阱、谢幕、装填五条核心体验都能被实际感知。
- 卡牌间联动已形成基础可玩循环。
- 战斗中无明显崩溃或大面积描述与行为不一致问题。

## 13. 下一阶段接口预留

为了减少后续返工，本阶段应预留以下扩展点：

- 稀有与高费 Build 卡牌扩展入口
- 后续遗物与药水对 20 张 MVP 卡的联动入口
- 灵魂卡替换基础卡的接入点
- 平衡统计与战斗记录扩展入口

这部分只要求结构可扩展，不要求本阶段实现完整内容。
