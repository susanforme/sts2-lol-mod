# Jhin MVP 文档 - 阶段 6：谢幕系统

## 1. 阶段目标

本阶段目标是实现烬的强力终结机制：谢幕。

本阶段重点是打通“满足终结条件 -> 使用谢幕类卡牌 -> 四次随机狙击 -> 残血增伤 -> 强反馈演出”的完整链路，让谢幕成为烬在残局和收割阶段的标志性输出手段。

## 2. 本阶段交付内容

必须交付以下内容：

1. 实现 `谢幕`。
2. 建立谢幕类卡牌的共同判断逻辑。
3. 增加基础演出反馈。

本阶段 `谢幕` 的要求：

- 只能在子弹为 0 时使用。
- 随机攻击 4 次。
- 敌人生命越低，伤害越高。
- 消耗。

共同判断逻辑要求：

- 子弹为空。
- 或本回合触发过华彩。

演出反馈要求：

- 屏幕压暗。
- 四道狙击线。
- 第四次射击音效更重。

## 3. 功能需求

## 3.1 基础谢幕卡牌

需要完成：

- 实现基础卡牌 `谢幕`。
- 该牌在满足条件时可以正常使用。
- 该牌打出后随机进行 4 次攻击。
- 该牌具有消耗属性。

最低要求：

- 每次使用稳定执行 4 段攻击。
- 每段攻击能正确选择目标。
- 使用后按消耗牌规则离场，不回到正常循环。

## 3.2 使用条件判断

需要完成：

- `谢幕` 只能在 `bullets == 0` 时使用。
- 谢幕类卡牌应具备统一的条件判定入口。
- 该入口需要支持两类条件：
  - 子弹为空
  - 本回合触发过华彩

本阶段最低要求：

- 基础 `谢幕` 先严格使用“子弹为空”条件。
- 共同判断结构必须能扩展到未来的“本回合触发过华彩后可用”类谢幕牌。
- 不允许把判断逻辑写死在单张卡里，导致后续难以复用。

## 3.3 四次随机攻击

需要完成：

- `谢幕` 在一次使用中执行 4 次独立攻击。
- 每次攻击随机选择敌人。
- 各段攻击都要参与残血伤害倍率计算。

最低要求：

- 四段攻击顺序稳定。
- 目标死亡后后续攻击应能继续对剩余敌人生效。
- 不会因为多段随机攻击而破坏战斗流程或造成崩溃。

## 3.4 残血增伤规则

需要完成：

- 敌人生命越低，受到的谢幕伤害越高。
- 每段攻击都应基于当前目标当前生命状态独立计算倍率。

参考目标：

- 满血：`1.0x`
- 半血：`1.5x`
- 低至 10% 血：`1.9x`

最低要求：

- 低血目标受到的伤害必须明显高于高血目标。
- 伤害倍率必须随目标剩余生命下降而上升。
- 不允许对非谢幕攻击误应用该倍率。

## 3.5 谢幕类卡牌共同判断逻辑

需要完成：

- 建立谢幕类卡牌共用的条件判断结构。
- 条件结构至少支持：
  - `子弹为空`
  - `本回合触发过华彩`
- 未来不同谢幕牌可按需要选择其中一种或组合使用。

最低要求：

- 基础 `谢幕` 使用该结构中的“空弹匣”规则。
- 后续如 `谢幕余音` 等卡牌可接入同一判断体系。

## 3.6 演出反馈

需要完成：

- 使用谢幕时屏幕压暗。
- 显示四道狙击线。
- 四次射击分别具备射击反馈。
- 第四次射击音效比前三次更重。

最低要求：

- 玩家能明显感知“谢幕”和普通攻击不是同一级别的动作。
- 四次攻击的节奏必须清楚，不允许演出与伤害结算完全脱节。
- 即使当前是占位表现，也要保证节奏感和识别度。

## 4. 建议实现的类与入口

本阶段建议优先完善以下类或接口：

- `谢幕` 卡牌类
- 谢幕条件判断基类或接口
- 多段随机攻击 Action
- 谢幕残血倍率计算入口
- 战斗演出或 UI 反馈入口

## 4.1 谢幕卡牌类

职责建议：

- 定义卡牌基础信息与消耗属性。
- 调用统一的谢幕条件判断。
- 触发谢幕多段攻击逻辑。

本阶段最低要求：

- 自身逻辑清晰。
- 不重复承载过多底层判断与多段攻击细节。

## 4.2 谢幕条件判断入口

职责建议：

- 统一处理谢幕类卡牌的可用性判断。
- 支持空弹匣条件和华彩条件。

本阶段最低要求：

- 与普通射击牌可用性判断清晰分离。
- 不影响已有射击牌、华彩牌逻辑。

## 4.3 多段随机攻击 Action

职责建议：

- 顺序执行 4 次随机攻击。
- 每次攻击独立选目标与计算伤害。
- 与演出反馈同步。

本阶段最低要求：

- 多段攻击链路集中管理。
- 不把 4 次攻击逻辑硬写在卡牌 `use()` 中。

## 4.4 残血倍率计算入口

职责建议：

- 根据目标当前生命百分比计算伤害倍率。
- 只服务于谢幕系卡牌。

本阶段最低要求：

- 公式清晰。
- 结算稳定。
- 不污染普通攻击或华彩攻击逻辑。

## 5. 与现有系统的兼容要求

需要确保谢幕系统不会破坏已完成的普通射击与华彩主链路。

必须满足：

- 谢幕不是普通射击牌时，不应错误消耗子弹。
- 谢幕的可用性判断不能覆盖普通射击牌逻辑。
- 谢幕的多段攻击不能误触发普通射击华彩判定。
- 谢幕的残血伤害倍率不能污染其他攻击牌。

## 6. 非目标

以下内容不属于本阶段：

- 其他高级谢幕牌的完整实现
- 谢幕与标记、陷阱、灵魂卡的复杂联动强化
- 最终品质全屏特效与完整音频资源
- 极端情况下的全量平衡调优
- 单元测试

说明：

- 本阶段优先把基础 `谢幕` 跑通，并为后续谢幕系卡牌预留公共结构。

## 7. 推荐最小实现策略

建议按以下顺序实现：

1. 先完成谢幕条件判断入口。
2. 完成基础 `谢幕` 卡牌的可用性与消耗属性。
3. 实现四段随机攻击 Action。
4. 接入残血增伤倍率。
5. 最后补屏幕压暗、狙击线和第四枪重音演出。

重点是先保证终结逻辑稳定，再增强表现力。

## 8. 验收标准

本阶段完成后，必须满足以下验收条件：

1. 子弹不为 0 时不能使用基础谢幕。
2. 子弹为 0 时可以正常使用。
3. 低血敌人受到更高伤害。
4. 谢幕不会破坏普通射击 / 华彩逻辑。

建议补充手动验证场景：

1. 当前子弹大于 0 时，`谢幕` 应显示不可用。
2. 当前子弹为 0 时，`谢幕` 应可正常打出。
3. `谢幕` 打出后应稳定执行 4 次随机攻击。
4. 对比满血目标和残血目标，残血目标应受到更高伤害。
5. 使用 `谢幕` 前后，普通射击牌的消耗子弹和华彩判定不应出现异常。
6. 第四次谢幕射击应在反馈上明显重于前三次。

## 9. 完成定义

当以下条件全部满足时，可判定“阶段 6 完成”：

- 基础 `谢幕` 已可用。
- 谢幕条件判断公共结构已建立。
- 四段随机攻击与残血倍率已正确实现。
- 基础演出反馈已接入。
- 与普通射击、华彩系统兼容，无明显冲突。

## 10. 下一阶段接口预留

为了减少后续返工，本阶段应预留以下扩展点：

- “本回合触发过华彩后可用”的谢幕变体入口
- 谢幕与标记、易伤、虚弱联动的扩展点
- 指定最低生命目标优先攻击的谢幕辅助入口
- 更高级的镜头、音效、屏幕反馈扩展点

这部分只要求结构可扩展，不要求本阶段实现完整内容。
